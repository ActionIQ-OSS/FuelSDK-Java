1 Salesforce Marketing Cloud Java SDK
2 ===================================
3 
4 The Salesforce Marketing Cloud Java SDK enables developers to easily
5 access the Salesforce Marketing Cloud (formerly ExactTarget) via the
6 Java platform. Among other things, the SDK:
7 
8 * automatically acquires and refreshes Marketing Cloud
9  access tokens
10 
11 * enables developers to access both Marketing Cloud SOAP
12  and REST APIs in the same session
13 
14 * exposes simplified versions of the most commonly used Marketing
15  Cloud objects and methods as Java native objects
16 
17 * provides passthroughs so developers can access the full
18  REST and SOAP APIs directly when they need to go beyond
19  the simplified interfaces
20 
21 * adds "sugar" methods for the most commonly used Marketing
22  Cloud features that make it easy to use those features (e.g.,
23  the SDK provides a SQL-like interface to data extensions)
24 
25 For more information about the Java SDK and how to use it, please see
26 the Javadocs at http://salesforce-marketingcloud.github.io/FuelSDK-Java/.
27 
28 Installation
29 ------------
30 
31 The easiest way to install the Java SDK is via Maven&mdash;simply add the following dependency to your project's `pom.xml`:
32 
33  <dependency>
34  <groupId>com.exacttarget</groupId>
35  <artifactId>fuelsdk</artifactId>
36  <version>1.1.0</version>
37  </dependency>
38 
39 Maven will automatically resolve, download, and install all dependencies for you.
40 
41 You can also download a jar file from the [Releases](https://github.com/salesforce-marketingcloud/FuelSDK-Java/releases) page or clone the repository and build a jar file yourself in the standard way. If you go this route, you'll need to ensure you have manually downloaded and installed all dependencies ([Apache CXF](http://cxf.apache.org), [Apache Commons BeanUtils](http://commons.apache.org/proper/commons-beanutils), [Apache log4j 1.x](http://logging.apache.org/log4j/1.2/), and [Google Gson](https://code.google.com/p/google-gson)) to your class path.
42 
43 Once you have the SDK installed, you'll need to obtain a client ID and client secret from App Center and place them in `fuelsdk.properties` using `src/main/resources/fuelsdk.properties.template` as a starting template. Theses values authenticate you to the Saleforce Marketing Cloud API. Please see https://code.exacttarget.com/getting-started/setting-your-development-environment for more information about how to use App Center to get a client ID and client secret.
